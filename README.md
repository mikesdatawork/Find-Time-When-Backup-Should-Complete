## About-Process

<p>Here's a quick script you can use to see exactly when ( or estimated time according to sql server ) that a backup, or restore operation will complete. This will give you a good indication of a certain time ( with more precision ) when a backup is set to complete. I use this and incorporate it into various other dependent jobs to execute at or around the time when these operations would finish.</p>      


## SQL-Logic
```SQL
use master;
set nocount on
select
'command' = sder.command
, 'percent_complete' = convert(numeric(6,2),sder.percent_complete)
, 'completion_time' = convert(char, dateadd(n,(sder.estimated_completion_time /60/1000),getdate()), 9) from
sys.dm_exec_requests sder
where
command in ('backup database', 'backup log', 'restore database')
```
